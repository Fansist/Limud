1. WHAT IS LIMUD?
================================================================================

Limud (Hebrew: למד, "learn") is an all-in-one AI-powered educational platform
for K-12 schools. It serves four user roles — Students, Teachers, Parents, and
Administrators — within a district-based multi-tenant architecture.

Core mission: Personalize education using AI tutoring, automated grading,
adaptive learning paths, cognitive science (spaced repetition, interleaving),
and gamification — all while meeting COPPA and FERPA compliance requirements
for student data privacy in the United States.

The platform is designed to be used by:
  - Individual homeschool families (single parent + children)
  - Small private schools (1 school, few teachers)
  - Large school districts (multiple schools, hundreds of teachers & students)

Production URL: https://limud.co
Repository:     https://github.com/Fansist/Limud
Package Name:   co.limud.app (Android APK)

2. TECH STACK & ARCHITECTURE
================================================================================

FRONTEND
  - Next.js 14.2.21 (App Router, Server Components)
  - React 18.3.1
  - TypeScript 5.7.2
  - Tailwind CSS 3.4.17
  - Framer Motion (animations)
  - Lucide React (icons)
  - Recharts (charts/analytics)
  - Zustand 5.0.2 (client state management)

BACKEND
  - Next.js API Routes (App Router, /api/*)
  - NextAuth.js 4.24.11 (JWT-based authentication, credentials provider)
  - Prisma 5.22.0 (ORM)
  - PostgreSQL (database)

AI
  - Google Gemini 2.5 Flash (via @google/genai SDK)
  - Fallback demo responses when no API key is set

EMAIL
  - Resend (transactional emails: password reset, weekly digests)

BUILD & DEPLOY
  - Node.js 20.x
  - PM2 (process manager)
  - Render.com (primary hosting)
  - cPanel (alternative hosting)
  - Bubblewrap (Android TWA APK generation)

ARCHITECTURE PATTERN:
  Next.js App Router with server-side rendering (SSR) and client components.
  The "/" page checks session and redirects logged-in users to their dashboard.
  All authenticated pages are behind middleware RBAC (role-based access control).
  API routes handle all data mutations and AI calls server-side.
  Demo mode runs entirely without a database using in-memory demo data.

/src — SOURCE CODE

src/middleware.ts
  THE edge middleware. Runs on EVERY request before the handler.
  Responsibilities:
    1. Block known malicious scanners (sqlmap, nikto, etc.)
    2. Block path traversal, SQL injection, XSS in URLs
    3. Block attack file extensions (.php, .asp, .exe, etc.)
    4. Block oversized URLs (>8192 chars)
    5. Edge-level rate limiting (500 req/min global, 30 req/min auth)
    6. Allow public paths without auth (/, /login, /register, etc.)
    7. Pass through static assets (/_next, .ico, .png, .css, .js)
    8. JWT authentication check (via next-auth/jwt getToken)
    9. Role-based access control (ADMIN, TEACHER, STUDENT, PARENT paths)
    10. Add OWASP security headers (HSTS, CSP, X-Frame-Options, etc.)

src/app/layout.tsx
  Root layout. Sets metadata, viewport, favicon, Open Graph tags.
  Reads 'limud-locale' cookie for i18n (Hebrew = RTL dir="rtl").

src/app/page.tsx

5. USER ROLES & ACCESS CONTROL (RBAC)
================================================================================

  STUDENT
    Can access: /student/*
    Features: AI tutor, assignments, focus mode, study planner, exam sim,
              forums, study groups, knowledge analytics, platforms, messages
    Special: Can request to join a district (/student/link-district)

  TEACHER
    Can access: /teacher/*
    Features: Create assignments, AI grading, quiz generator, analytics,
              student management, reports, lesson planner, intelligence,
              resource exchange, content library, forums, messages
    Special: Homeschool parents get teacher access too

  PARENT
    Can access: /parent/*
    Features: Children overview, weekly reports, messages, goal tracking
    Special: Homeschool parents also get teacher-level access

  ADMIN (District Administrator)
    Can access: /admin/*
    Features: District management, schools, classrooms, user management,
              announcements, analytics, billing/payments, security,
              audit logs, bulk provisioning, link request approvals
    Permissions per admin: canCreateAccounts, canManageSchools,
                           canManageBilling, canViewAllData, canManageClasses

  MASTER DEMO
    Email: master@limud.edu / Password: LimudMaster2026!
    Has isMasterDemo=true in JWT — bypasses ALL role checks
    Can access every page across all roles (for demonstration purposes)

  MIDDLEWARE ENFORCEMENT:
    - Edge middleware runs on every request
    - Checks JWT token via next-auth/jwt
    - Maps URL paths to required roles
    - Returns 401 for API, redirects to /login for pages
    - Public paths bypass auth entirely

6. FEATURE CATALOG — WHAT EVERY FEATURE DOES
================================================================================

STUDENT FEATURES:
─────────────────

  1. AI TUTOR (src/app/student/tutor, /api/tutor)
     What it does: Chat-based AI tutoring powered by Gemini 2.5 Flash.
     The tutor NEVER gives direct answers — it uses the Socratic method to
     guide students. It's personalized using the student's survey data
     (hobbies, favorite games/movies, dream job, learning style).
     Data: AITutorLog table stores full conversation history.

  2. ASSIGNMENTS (src/app/student/assignments, /api/assignments, /api/submissions)
     What it does: Students view assigned work, submit answers (text or file),
     receive AI-generated feedback and scores. Supports adaptive assignments
     that modify content per student's learning style.

  3. FOCUS MODE (src/app/student/focus, /api/focus)
     What it does: Pomodoro-style focus timer. Students select subject/topic,
     set timer (default 25 min), track distractions. Records focus score.

  4. STUDY PLANNER (src/app/student/study-planner, /api/study-planner)
     What it does: AI-recommended study schedule. Creates sessions based on
     upcoming assignments, weak skills (from spaced rep), and learning DNA
     peak hours. Students can mark sessions complete.

  5. EXAM SIMULATOR (src/app/student/exam-sim, /api/exam-sim)
     What it does: AI generates practice exams by subject/grade. Timed tests
     with auto-grading. Shows strengths/weaknesses and predicted real exam score.

  6. KNOWLEDGE ANALYTICS (src/app/student/knowledge, /api/skills, /api/learning-dna)
     What it does: Visual dashboard of skill mastery, learning DNA profile,
     mistake patterns, confidence tracking, spaced repetition schedule.

  7. FORUMS (src/app/student/forums, /api/forums)
     What it does: Course-based discussion boards. Students ask questions,
     upvote answers, teachers can endorse correct answers.

  8. STUDY GROUPS (src/app/student/study-groups, /api/study-groups)
     What it does: Create/join study groups. Chat with group members.
     Share resources. AI can summarize group discussions.

  9. STUDENT INTEREST SURVEY (src/app/student/survey, /api/survey)
     What it does: Collects student interests (hobbies, favorite games/movies/
     books, dream job, learning style, motivators, challenges). This data is
     fed to the AI tutor to personalize responses with relatable analogies.

  10. CLASSROOMS (src/app/student/classrooms, /api/student/classrooms)
      What it does: View enrolled classrooms, teachers, classmates.

  11. MESSAGES (src/app/student/messages, /api/messages)
      What it does: Direct messaging with teachers and parents.

  12. PLATFORMS (src/app/student/platforms, /api/platforms)
      What it does: Connect external learning platforms (Google Classroom, etc.)

  13. GROWTH (src/app/student/growth)
      What it does: Long-term progress tracking, growth charts, trend analysis.

  14. JOIN DISTRICT (src/app/student/link-district, /api/district-link)
      What it does: Students without a district can search for and request to
      join one. Admin approves/denies. Links student to district + school.

TEACHER FEATURES:
─────────────────

  15. DASHBOARD (src/app/teacher/dashboard)
      What it does: Overview of classes, pending submissions, quick actions.

  16. ASSIGNMENT CREATION (src/app/teacher/assignments, /api/assignments)
      What it does: Create assignments with type, rubric, difficulty, skills,
      due date. Supports adaptive mode (AI adapts per student learning style).

  17. AI GRADING (src/app/teacher/grading, /api/grade)
      What it does: AI grades submissions against rubric. Shows score,
      strengths, improvements, encouragement. Teacher can override.
      Detects misconceptions automatically.

  18. QUIZ GENERATOR (src/app/teacher/quiz-generator, /api/quiz-generator)
      What it does: AI generates quizzes by subject, grade, topic, difficulty,
      question count. Saves as reusable templates. Can convert to assignments.

  19. ANALYTICS (src/app/teacher/analytics, /api/analytics)
      What it does: Class-wide score distributions, skill mastery heatmaps,
      trending topics, at-risk student identification.

  20. STUDENT MANAGEMENT (src/app/teacher/students)
      What it does: View individual student profiles, progress, learning DNA,
      skill records, recent submissions.

  21. REPORTS (src/app/teacher/reports, /api/reports, /api/teacher/reports)

8. AI SYSTEM (GEMINI 2.5 FLASH)
================================================================================

MODEL: gemini-2.5-flash (configurable via AI_MODEL env var)
SDK:   @google/genai
KEY:   GEMINI_API_KEY or GOOGLE_API_KEY

HOW AI IS USED:
  1. AI TUTOR — Socratic method tutoring, personalized via student survey data.
     System prompt: Never give direct answers, use hints and questions.
     Temperature: 0.7, Max tokens: 800.

  2. AI GRADING — Evaluates submissions against rubric.
     Returns JSON: {score, maxScore, feedback, strengths, improvements}.
     Temperature: 0.3, Max tokens: 1024.

  3. QUIZ GENERATION — Creates questions by subject/grade/difficulty.
     Returns JSON array of questions with options, answers, explanations.

  4. REPORT GENERATION — Student progress reports with projections.
     Returns JSON: {summary, strengths, areasForGrowth, recommendations,
     parentNote, gradeProjection, behavioralNotes, nextSteps}.
     Temperature: 0.5, Max tokens: 1500.

  5. CURRICULUM ANALYSIS — Skill gap analysis and pacing recommendations.
     Temperature: 0.4, Max tokens: 1500.

  6. WRITING FEEDBACK — Detailed writing evaluation across 5 categories
     (mechanics, organization, voice, content, creativity).
     Temperature: 0.3, Max tokens: 1500.

  7. ASSIGNMENT ADAPTATION — Transforms assignments per student learning style
     (visual, auditory, ADHD-friendly, structured, kinesthetic).

  8. MATH SOLVER — Step-by-step problem solving with error detection.

  9. CONCEPT MAPS — Generates nodes and edges for topic visualization.

  10. LESSON PLANS — Generates complete lesson plans with activities.

DEMO MODE:
  When GEMINI_API_KEY is not set or is "demo-mode", all AI functions
  fall back to pre-written template responses. The demo responses are
  context-aware (e.g., math questions get math-themed responses).
  Quiz generation uses extensive question banks in ai-generators.ts
  with hundreds of real educational questions.

ERROR HANDLING:
  - Invalid API keys are detected before calling the API (fail fast)
  - Auth errors, quota limits, safety filters all have specific messages
  - callGeminiSafe() returns {ok, data} or {ok: false, error} without throwing
  - extractJSON() handles markdown fences, trailing commas, partial responses

9. SECURITY & COMPLIANCE (COPPA / FERPA / OWASP)
================================================================================

COPPA (Children's Online Privacy Protection):
  - ParentalConsent model tracks consent per child + type
  - Consent types: data_collection, ai_interaction, third_party_sharing
  - Minimal data collection for minors
  - /api/security/consent endpoints for managing consent

FERPA (Family Educational Rights and Privacy Act):
  - DataAccessLog tracks every access to student education records
  - SecurityAuditLog provides tamper-proof audit trail (7-year retention)
  - DataDeletionRequest supports right-to-delete
  - Role-based access control restricts data visibility

OWASP Top 10 Mitigations:
  - Injection: Input sanitization, parameterized queries (Prisma)
  - Broken Auth: Brute-force protection, progressive lockout, bcrypt
  - XSS: CSP headers, input sanitization, React auto-escaping
  - IDOR: Auth checks on every API route, ownership validation
  - Security Misconfiguration: HSTS, X-Frame-Options, no server info leaks
  - CSRF: NextAuth CSRF tokens, SameSite cookies
  - File Upload: Size limits, type validation
  - Rate Limiting: Per-IP, per-user, per-endpoint sliding windows

RATE LIMITS:
  Edge level:  500 req/min global, 30 req/min auth
  App level:   60 req/min global, 5 login/min, 3 register/min,
               100 API/min, 10 AI/min, 5 upload/min, 20 PII/min

PASSWORD POLICY (NIST SP 800-63B):
  Min 10 chars, max 128, require uppercase + lowercase + number.
  Special characters not required (per NIST recommendation).

ENCRYPTION:
  PII fields encrypted with AES-256-GCM.
  Key derived from PII_ENCRYPTION_KEY or AUTH_SECRET.

SECURITY HEADERS (on every response):
  Strict-Transport-Security, X-Frame-Options: DENY,
  X-Content-Type-Options: nosniff, X-XSS-Protection,
  Content-Security-Policy, Referrer-Policy,
  Permissions-Policy, CORP, COOP

11. ADAPTIVE LEARNING & COGNITIVE SCIENCE
================================================================================

SM-2 SPACED REPETITION:
  Implements SuperMemo SM-2 algorithm in cognitive-engine.ts.
  Quality: 0-5 scale (0=complete failure, 5=perfect).
  Correct (quality >= 3): interval grows exponentially via ease factor.
  Incorrect: reset to interval=1.
  Ease factor minimum: 1.3.
  Used in: SkillRecord, SpacedRepItem, VocabEntry.

ADAPTIVE DIFFICULTY:
  Targets 70-85% success rate (optimal learning zone).
  Difficulty levels: BEGINNER, EASY, MEDIUM, HARD, ADVANCED.
  AI adjusts assignment content per student's skill level.

LEARNING DNA PROFILE:
  Tracks over time: learning speed, retention rate, preferred modality,
  peak study hours, average session length, attention span.
  Modality detection: visual, auditory, kinesthetic, reading.
  Based on: video completion rate, reading speed, quiz vs project preference,
  tutor usage patterns.

ASSIGNMENT ADAPTATION:
  AdaptedAssignment model stores AI-transformed versions.
  Styles: visual, auditory, adhd_friendly, structured, kinesthetic,
          reading_writing.
  Includes: modifications, scaffolding, format changes, method suggestions.

STUDENT SURVEY (Personalization Input):
  Collects: favorite subjects, hobbies, books, movies, games,
           dream job, learning style, learning needs (ADHD, dyslexia, etc.),
           preferred formats, motivators, challenges.
  Used by: AI tutor for personalized analogies and examples.

17. KNOWN BUGS & ISSUES
================================================================================

  1. DEMO STATISTICS IN REAL ACCOUNTS
     Demo data can leak into real account views in some edge cases.
     The demo-detection logic (checking for "demo-" prefix in user ID)
     needs hardening across all API routes.

  2. DISTRICT CLASSROOM FILTERING
     src/app/api/district/classrooms.ts has districtId filtering that
     may prevent teachers or students from seeing classrooms in certain
     account type configurations.

  3. APP_VERSION MISMATCH
     Some files reference different versions (e.g., middleware says "11.0.0"
     in X-Limud-Version header while config.ts says "13.0").

  4. DIGITAL ASSET LINKS FOR APK
     The Android APK requires /.well-known/assetlinks.json on limud.co
     with the signing key's SHA-256 fingerprint. This has not been
     configured yet, which means the TWA may show the browser URL bar.

  5. DATABASE MIGRATIONS
     The project uses "prisma db push" rather than proper migrations.
     For production, should migrate to "prisma migrate" for safe schema changes.

  6. FILE UPLOAD STORAGE
     FileUpload model stores base64 in a Text field. For production scale,
     should move to object storage (S3, R2, etc.).

19. HOW TO CONTRIBUTE — GETTING STARTED
================================================================================

SETUP:
  1. Clone: git clone https://github.com/Fansist/Limud.git
  2. Install: npm install
  3. Copy env: cp .env.example .env
  4. Database: Set DATABASE_URL to a PostgreSQL instance
     (Use Neon, Supabase, or local PostgreSQL)
  5. Generate Prisma: npx prisma generate
  6. Push schema: npx prisma db push
  7. (Optional) Seed: npx prisma db seed
  8. Run: npm run dev → http://localhost:3000
  9. Use demo accounts or register a new one.

CODING CONVENTIONS:
  - TypeScript strict mode
  - Tailwind CSS for all styling (no custom CSS except globals.css)
  - App Router (src/app/) for all routes
  - API routes use GET/POST handlers with NextResponse
  - Prisma for all database access (never raw SQL)
  - AI calls go through src/lib/ai.ts (never call Gemini directly)
  - Demo mode must be supported in every feature
  - All user-facing text should use i18n translations

FILE NAMING:
  - Pages: src/app/{role}/{feature}/page.tsx
  - Layouts: src/app/{role}/{feature}/layout.tsx
  - API routes: src/app/api/{domain}/route.ts
  - Components: src/components/{category}/{ComponentName}.tsx
  - Libraries: src/lib/{name}.ts

ADDING A NEW FEATURE:
  1. Add database model(s) to prisma/schema.prisma
  2. Run: npx prisma generate && npx prisma db push
  3. Create API route(s) in src/app/api/{feature}/route.ts
  4. Create page(s) in src/app/{role}/{feature}/page.tsx
  5. Add navigation link in DashboardLayout.tsx (GROUPED_NAV)
  6. Add demo data in src/lib/demo-data.ts
  7. Add demo fallback in API route (check for "demo-" user IDs)
  8. Add i18n strings in messages/*.json
  9. Write tests in __tests__/
  10. Update this guide!

TESTING:
  npm test              — Run all tests
  npm run test:coverage — Run with coverage report
  npm run test:watch    — Watch mode

GIT WORKFLOW:
  - Main branch: main
  - Version bumps in: package.json, src/lib/config.ts (APP_VERSION)
  - Commit message format: "v{version}: {description} — Update {N}"
